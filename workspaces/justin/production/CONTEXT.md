# Production Context

## Purpose
Execution layer for production-ready deliverables and build artifacts.

## When to Use
Use for turning plans into concrete briefs, specs, builds, assets, and shipped outputs.

## Read First
- Active brief/spec in `briefs/` or `specs/`
- Prior build artifacts in `builds/` for continuity

## Output Types
Implementation briefs, technical specs, build artifacts, packaged outputs.

## Do Not Load By Default
Do not pull full research/content history unless directly required for the build.
