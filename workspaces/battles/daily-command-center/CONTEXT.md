# Daily Command Center Context

## Purpose
Operating area for Battles' daily priorities, open loops, schedule decisions, commitments, and near-term execution.

## When to Use
Use this workspace when Jarvis is planning the day, prioritizing tasks, reviewing commitments, deciding what to do next, or turning scattered inputs into a concrete action plan.

## Files
- `priorities.md` - current focus areas and priority stack.
- `open-loops.md` - unresolved tasks, questions, and follow-ups.
- `decisions-log.md` - day-to-day operating decisions worth remembering.

## Output Rules
- Keep daily plans short and actionable.
- Prefer the smallest next step when the user seems overloaded.
- Record durable daily decisions here, but do not duplicate database memory or private tokens.
