Enhance my existing Jarvis AI assistant to handle YouTube transcript retrieval using a cloud-first, no-VPS-required pipeline.

Context:
- Jarvis runs as a hosted backend (not on Replit)
- Claude orchestrator decides tool usage
- ChatGPT-4o-mini handles execution
- All processing should happen within the current backend environment
- VPS should NOT be required as a primary solution

-----------------------------------
GOAL
-----------------------------------
Achieve high success rate for YouTube transcript retrieval using layered fallbacks without relying on browser automation or VPS.

-----------------------------------
PIPELINE
-----------------------------------

Step 1: Detect YouTube URLs
- Automatically detect valid YouTube links in user messages

-----------------------------------

Step 2: Try Transcript API / Library
- Use any available YouTube transcript libraries or APIs
- Return transcript if available

-----------------------------------

Step 3: Try yt-dlp (primary method)
Run:

yt-dlp --skip-download --write-subs --write-auto-subs --sub-langs "en.*"

- Prefer manual subtitles first
- fallback to auto-generated subtitles
- convert output to plain text

-----------------------------------

Step 4: Audio Transcription Fallback
If no subtitles:

1. Extract audio:
   yt-dlp -f bestaudio --extract-audio --audio-format mp3

2. Convert to WAV:
   ffmpeg -i input.mp3 -ar 16000 -ac 1 output.wav

3. If long audio:
   - split into chunks (5–10 minutes)

4. Transcribe using:
   - OpenAI speech-to-text API (preferred)
   OR
   - local faster-whisper if available

5. Combine results into one transcript

6. Label output:
   "AI-generated transcript (no official captions available)"

-----------------------------------

Step 5: Tavily Fallback (optional)
If transcript fails:
- search for summaries or external transcripts
- return best available explanation

-----------------------------------

Step 6: Response Handling
- short transcript → send message
- long transcript → send .txt file
- very long → send summary + file

-----------------------------------

ERROR HANDLING
-----------------------------------
Handle:
- invalid URL
- private video
- age-restricted video
- no captions
- yt-dlp failure
- ffmpeg missing
- transcription failure
- timeout

Return clean user-friendly messages.

-----------------------------------

IMPORTANT RULES
-----------------------------------
- Do NOT rely on Playwright as primary method
- Do NOT require VPS
- Do NOT attempt to bypass login or private video protections
- Always degrade gracefully instead of failing silently

-----------------------------------

OUTPUT FORMAT
-----------------------------------
{
  success: true,
  method: "api | yt-dlp | transcription | tavily",
  transcript: "...",
  notes: "..."
}
